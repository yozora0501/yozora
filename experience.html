Guide First

作业日常

负责中压带电作业工程的落地与执行，涵盖从计划到归档的完整链条。

  • 计划前置：每日填写作业票、制定次日工作计划，编写作业流程与安全规范，并录入后台供服系统。
  • 现场执行：按工程计划前往指定地点，直接执行带电作业或担任现场监护，保障作业安全与合规。
  • 交付闭环：作业完成后对接交付，更新后台系统状态，整理归档工作票等工程文档，确保每个环节可追溯。

Execution Board

工作流程

这一板块集中展示我的执行能力：如何依据工作票推进现场作业、如何处理关键节点、以及如何让每个步骤都可核对、可复盘。

工作票体系：关键节点管控

  • 票面类型包括：勘察单、基础票面、作业计划、工程提交单。高风险作业额外增加人员管控、车辆管控、三措一案等风险措施。
  • 流程优化：重新规划编写流程，按区块拆分、统一编写单区块项目，使记忆锚点更清晰，工作票错误率降低30%。
  • 通过 Python 调用 Word 文本库，实现重复内容一次编写、多次复用，减少手动录入带来的差错。

工作推进流程：从任务到归档的完整链路

  • 接到任务 → 现场勘察 → 规划施工方案 → 提交流程与清单 → 编排人员 → 编写工作票 → 施工执行 → 项目归档。
  • 清晰的结构使项目安全有序推进，并能提前识别风险、制定预防措施。
  • 风险交底：作业前组织班前会，向全体成员交底风险点和操作要点；作业后召开班后会，复盘异常、记录改进点。
  • 跨部门同步：与调度、安监、运行等部门保持实时信息同步，及时获取停电计划、设备状态等关键信息，避免作业冲突。

文档整理

文档整理：把经验沉淀成方法

  • 按任务类型整理过程记录、关键判断和处理结果，避免信息碎片化。

  • 增加可复用模板，减少重复劳动并且可精准定位对应的作业流程。

项目归档：让成果可追溯、可复用

  • 按阶段整理输入材料、执行记录、结果产出和复盘结论。
  • 按工程时间规划梳理归档项目，并且整理成册，项目全流程全时间段可追溯。
